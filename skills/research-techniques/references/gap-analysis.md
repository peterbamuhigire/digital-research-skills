# Gap analysis

A complete-feeling research report can hide structural blind spots. This skill catalogues them.

## Three kinds of gap

| Kind | Cause | Fix |
|---|---|---|
| **Search gap** | We didn't search the right place | Wave-2 agent with new search targets |
| **Structural gap** | The data doesn't exist publicly | Acknowledge, scope-flag, propose primary research |
| **Scope gap** | We deliberately excluded a perspective | Acknowledge — note the exclusion in the report |

Don't treat all gaps the same. A search gap is fixable; a structural gap is a finding.

## Standard gap dimensions

For every research project, audit along these axes:

- **Geographic** — countries / cities / neighbourhoods missing
- **Cohort** — adjacent population not yet researched
- **Source type** — academic / regulatory / news / social tier imbalance
- **Time** — historical depth vs current snapshot
- **Language** — non-English-language sources rarely surfaced
- **Vulnerable groups** — single women, refugees, persons with disabilities, elderly, minors
- **Quantitative** — what % statistics are anchored vs anecdotal
- **Triangulation** — single-source claims still in the report
- **Verbatim quotes** — themes without primary-voice quotes

## Decision rules

- **Every research file ends with a "What's not yet mapped" section.** Living document discipline.
- **Each gap labelled by kind** (search / structural / scope).
- **Search gaps get a Wave-2 brief** — explicit search targets, source types, queries.
- **Structural gaps surface as report findings**, not as embarrassments. ("No East African student-hostel-owner P&L data exists in academic literature" is itself a finding worth highlighting.)
- **Scope gaps are restated each pass** — so they don't accidentally close in Wave 2.

## Wave-2 brief construction

A good gap-fill brief contains:

1. The Wave-1 themes already covered (DO NOT re-confirm)
2. The specific gaps to chase (numbered)
3. New source targets per gap
4. Hard constraints (any topics excluded by user)
5. Output shape — append to existing files, don't overwrite

## Anti-patterns

- Listing gaps without classifying them
- Re-running Wave 1 to "fill the gap" — produces duplicate sources
- Treating scope exclusions as gaps to fill
- Closing gap sections to make a report look "complete" when the data still doesn't exist
- Conflating "we didn't find it" with "it doesn't exist"

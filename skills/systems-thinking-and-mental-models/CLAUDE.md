# Claude-specific — systems-thinking-and-mental-models

- This skill is a router. Always read `SKILL.md` first to pick the right toolkit; do not load all four references by default.
- For "how does this system fit together / who does what to whom" → `references/systemigrams.md`. Write the SoI description as prose first; never draw before that.
- For "why does this keep happening / where would intervention work" → `references/causal-loops-and-leverage-points.md`. Identify the Iceberg level before recommending an intervention.
- For "what frame should I be using on this problem" → `references/mental-models-catalog.md`. Walk the catalog; record which models were applied.
- For "I am about to ship a probability or forecast" → `references/decision-science-heuristics.md`. State the base rate, the reference class, and the calibration approach.
- Never present a system map or causal loop diagram without the prose description that generated it.
- Every loop, every probability, every base rate is still a claim — `source-evaluation` evidence-discipline applies.
- Combine toolkits where appropriate: a regulatory-landscape map is often systemigram + causal loops; a forecast is mental models + decision science.

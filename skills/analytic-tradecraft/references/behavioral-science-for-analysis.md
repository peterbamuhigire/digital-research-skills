# Behavioral-science support for analysis

Use this reference when the analytic problem is being degraded by how the team learns, collaborates, evaluates, or communicates rather than by missing search results alone.

## Four operating domains

1. **Learning and evaluation.** Analysts improve faster when methods are explicit, feedback is regular, and post-mortems are tied to judgment quality rather than only final outcomes.
2. **Analysis.** Basic decision analysis, probability handling, and statistical familiarity are not specialist luxuries; they are baseline tradecraft.
3. **Collaboration.** Team structure should reduce unchallenged consensus, not simply speed up coordination.
4. **Communication.** A strong judgment can still fail if uncertainty, assumptions, and alternatives are not communicated in a way decision-makers can act on.

## How to use it

- If a judgment failure came from weak reasoning, run ACH / KAC / bias audit.
- If it came from weak team process, document what collaboration pattern failed.
- If it came from weak presentation to the reader, repair the communication layer before shipping.
- Treat training gaps as operational risk, not HR trivia.

## Minimum analyst baseline

- Comfort with simple probability language and confidence intervals
- Ability to distinguish descriptive data from inference
- Ability to explain assumptions, alternatives, and indicators
- Ability to work in a team without suppressing dissent

## Anti-patterns

- Treating analytic quality as an individual talent problem only
- Confusing consensus with rigor
- Letting communication polish hide unresolved assumptions
- Using statistics decoratively without understanding what they support

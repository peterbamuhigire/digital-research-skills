# mind-mapping-and-synthesis

Buzan-method mind mapping adapted for analyst synthesis, with Mermaid `mindmap` blocks as the engine-native artifact.

## Files

- `SKILL.md` — entry, when to use, seven steps + ten Laws, reference router
- `CLAUDE.md` — Claude-Code-specific load order
- `AGENTS.md` — Codex / generic-agent equivalent
- `references/mind-map-construction.md` — Buzan's method applied to analyst use cases (synthesis, taxonomy, brief decomposition)
- `references/study-and-recall-techniques.md` — preview/active/synthesis/spaced-revision routine for learning new domains
- `references/mermaid-mindmap-patterns.md` — concrete Mermaid templates for cohort synthesis, pain-point taxonomy, regulatory landscape, research-plan decomposition

## Source books

- Tony Buzan, *Mind Map Mastery — The Complete Guide to Learning and Using the Most Powerful Thinking Tool in the Universe*
- Tony Buzan, *Buzan Study Skills Handbook* (image-only EPUB; method drawn from *Mind Map Mastery* which covers the same content)

## Hard rule

Mind maps are a *thinking* artifact. Where the audience expects prose, re-render. Where the structure is sequential, dynamic, or temporal, use a different tool (`systems-thinking-and-mental-models` for dynamics; standard outlines for sequences).

Every fact placed on a branch is still subject to `source-evaluation` — the radial form does not exempt claims from evidence discipline.

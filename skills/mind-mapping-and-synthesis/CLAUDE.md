# Claude-specific — mind-mapping-and-synthesis

- Use this skill when the corpus has hierarchical / radial structure (synthesis, taxonomy, landscape, brief decomposition). Use a different tool when the structure is sequential, dynamic, or temporal.
- Always start from a specific centre (research question, deliverable, system, cohort). If the centre is hard to choose, the mind map is premature — refine the brief first.
- Apply Buzan's discipline: one keyword per branch, 5–6 main branches, sub-branches that earn their place. Sentences on branches are an anti-pattern.
- For engine output, render as Mermaid `mindmap` blocks. Cross-links go in a separate prose block beneath the mindmap (Mermaid `mindmap` does not natively support cross-edges).
- Build the first mind map quickly. Iterate — the redrawing is the thinking.
- For a *learning* task (analyst new to a domain), follow the preview / active reading / synthesis / spaced-revision routine in `references/study-and-recall-techniques.md`.
- Where the audience needs prose, treat the mind map as upstream — generate it, then re-render. Don't ship the mind map as the final artifact unless the audience is asking for one.
- Every fact on the map is still a claim — `source-evaluation` evidence-discipline applies.

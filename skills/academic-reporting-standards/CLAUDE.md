# Claude-specific — academic-reporting-standards

- **Trigger.** Invoke this skill on any output intended for an examination committee, a peer-reviewed journal, an institutional review board, or a funder. Doctoral dissertations, master's theses, journal articles, systematic reviews, conference papers, research proposals, grant applications, book proposals.
- **Read order.** SKILL.md first; then load the references the decision router specifies.
- **Hard rule — genre.** Use neutral topic headings (Introduction, Literature Review, Methodology, Findings, Discussion, Conclusion). Do **not** use action titles in academic chapters; that is `executive-communication` discipline for a different audience.
- **Hard rule — original contribution.** Every doctoral artefact states explicitly, in the abstract and in chapter 1, what is new. Frame as gap-filling / inconsistency-resolving / conflict-clarifying / theory-testing / theory-building.
- **Hard rule — methodology.** Justify every choice. Tie each justification back to the theory or phenomenon under investigation. A methodology chapter that describes without justifying is incomplete.
- **Hard rule — reporting standard.** Match the EQUATOR-network checklist to the study type and submit it with the artefact. PRISMA for systematic reviews, CONSORT for RCTs, STROBE for observational, MOOSE for meta-analysis of observational, GRADE for evidence quality.
- **Hard rule — bias acknowledgement.** Discussion section acknowledges biases and the bases for them; accommodates alternative perspectives; avoids polemics. (Brause, Ch 14.)
- **Refuse to embellish.** Findings are calibrated to data. Over-claiming and under-claiming are both examiner red flags. If the data don't support a stronger claim, the claim doesn't ship.
- **Citation style.** Match the institution or journal. Bluebook (US legal) / OSCOLA (Oxford legal) / Chicago / Turabian (humanities) / APA (social sciences) / AMA / Vancouver (medicine) / MLA (literature). Confirm with user if not specified.

See `SKILL.md`.
